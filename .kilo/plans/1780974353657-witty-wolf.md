# Plan: Performance Optimization + Chinese Font Fix

## Issue 1: Core Pipeline Performance

### Bottleneck Analysis

**A. `research_arbitrage.py` — `backtest_curve_arbitrage()` (lines 188-323) [SEVERE]**
- Pure Python row-by-row loop over 5-minute bars per symbol per day
- For 21 symbols × ~1200 trading days × ~50 bars/day = ~1.26M iterations in Python
- Each iteration does spread computation, z-score, position logic
- **Fix**: Rewrite inner bar loop as numba JIT (`@numba.njit`), pre-extract arrays

**B. `test_alt_alphas.py` — `fast_newey_west_t()` (lines 71-102) [MODERATE]**
- Called hundreds of times per symbol×factor×horizon combination
- Inner double loop over n × lags for HAC covariance matrix
- **Fix**: Convert to `@numba.njit(cache=True)` — pure numpy math, no pandas

**C. `evaluate_alpha.py` — capacity decay rolling calculations (lines 186-190) [MODERATE]**
- `unstack().rolling(20).std().stack()` and `unstack().rolling(20).mean().stack()` on multi-index DataFrames
- These cause massive intermediate DataFrame creation
- **Fix**: Compute rolling std/mean per-symbol using numpy, avoid unstack/stack round-trip

**D. `alphas.py` — `compute_alphas()` — data loading [MINOR]**
- Each of 23 symbols loads and processes independently
- Spot basis data re-loaded from parquet per symbol (already loaded once, but per-symbol filtering is redundant)
- Kalman filter already uses numba but input is float64; casting to float32 earlier reduces memory bandwidth
- **Fix**: Cast to float32 before Kalman filter; batch-load parquets once

### Implementation Steps

#### Step 1: Numba-accelerate `fast_newey_west_t` in `test_alt_alphas.py`
- Add `@numba.njit(cache=True)` to new function `_numba_newey_west_t(x, y, lags)`
- Input arrays are 1D float64; no pandas objects inside
- Wrap with fallback in `fast_newey_west_t()` that calls the numba version
- Expected speedup: 10-50x for this function

#### Step 2: Numba-accelerate `backtest_curve_arbitrage` inner loop in `research_arbitrage.py`
- Extract aligned 5-min bar arrays (near_close, dom_close) per day
- Write `@numba.njit` function that takes: near_prices, dom_prices, mean_s, std_s, entry_z, exit_z, tc_rate
- Returns day_pnl as float
- Replace lines 266-318 inner loop with numba call
- Expected speedup: 50-200x for the hot path

#### Step 3: Vectorize capacity decay rolling calculations in `evaluate_alpha.py`
- Replace `df["vol_20"] = df["returns"].unstack().rolling(20).std().stack()` pattern
- Compute per-symbol using numpy: `np.lib.stride_tricks.sliding_window_view` or grouped apply
- Same for `market_notional_volume` rolling mean
- Cast intermediate results to float32
- Expected speedup: 3-10x for this section

#### Step 4: fp32 optimization in `alphas.py`
- Cast price/basis data to float32 early in `compute_alphas()` per-symbol loop
- Kalman filter already returns float64 — keep that (numba jit), but cast input to float32 where precision is not critical
- Rolling operations (rolling(12).mean, rolling(20).skew, etc.) operate on float64 — cast to float32 after computation where appropriate
